# Лабораторная работа №1. Модель завода по производству мебели

Данный проект представляет собой программную систему для моделирования работы мебельной фабрики.  
Она управляет персоналом, инструментами, материалами и полным циклом производства мебели — от принятия заказа до доставки клиенту.

---

## Предметная область
Система моделирует:
- Клиентов, делающих заказы на мебель
- Рабочих с разными специализациями
- Инструменты с ограниченной прочностью
- Склад материалов и склад готовой продукции
- Цех для сборки и хранения готовых изделий
- Этапы производства мебели с проверкой качества, упаковкой и доставкой

---

## Основные сущности
- **Furniture** — объект мебели с типом, материалами и статусом (`FurnitureState`)  
- **Material** — базовый класс материалов, включая `Wood` и `Metal`  
- **Worker** — рабочий с определенной специализацией  
- **Customer** — клиент, делающий заказ  
- **Tool** — инструмент с ограниченной прочностью (`durability`)  
- **Warehouse** — склад для хранения материалов и готовой продукции  
- **Workshop** — цех для хранения и учета готовой мебели  
- **Factory & Operation** — логика обработки мебели через операции:
  - `PreparationOperation` — подготовка материалов
  - `CreateElementOperation` — изготовление деталей
  - `AssemblyOperation` — сборка
  - `CheckOperation` — контроль качества
  - `PackingOperation` — упаковка
  - `DeliveryOperation` — доставка клиенту

---

## Статусы мебели (`FurnitureState`)

| Статус | Описание |
|--------|----------|
| CREATED | Новый заказ |
| MATERIALS_PREPARED | Материалы подготовлены |
| ELEMENTS_MANUFACTURED | Детали изготовлены |
| ASSEMBLED | Мебель собрана |
| QUALITY_CHECKED | Пройдена проверка качества |
| PACKED | Упакована |
| STORED | На складе |
| DELIVERED | Доставлена клиенту |

## Технические особенности
- Python 3.10+ с аннотациями типов  
- Собственные исключения: `InvalidDataError`, `InvalidAmountError`, `InvalidOperation`  
- CLI-интерфейс для управления фабрикой  
- Сохранение и загрузка состояния фабрики в JSON  
- Модульная архитектура: `people`, `furniture`, `tool`, `warehouse`, `workshop`, `operations`, `factory`

---

## Функционал

### Меню
1. Новый клиент и заказ  
2. Производство мебели  
3. Просмотр склада готовой продукции  
4. Просмотр всех клиентов  
5. Просмотр всех работников  
6. Просмотр всех инструментов  
7. Выход с автоматическим сохранением

### Этапы производства
1. **Подготовка материалов** — проверка и списание металла и дерева со склада  
2. **Изготовление деталей** — создание всех элементов мебели  
3. **Сборка** — объединение деталей в готовый объект  
4. **Контроль качества** — проверка на дефекты, возможный откат к этапу производства  
5. **Упаковка** — выбор упаковочного материала и фиксация упаковщика  
6. **Доставка** — назначение курьера, фиксирование адреса и времени доставки, помещение на склад готовой продукции

## Структура проекта
- `models/furniture.py` — `Furniture` и `FurnitureState`  
- `models/people.py` — `Worker` и `Customer`  
- `models/tool.py` — класс `Tool`  
- `models/warehouse.py` — склад материалов  
- `models/workshop.py` — сборочный цех  
- `models/factory.py` — фабрика и управление операциями  
- `operations/operations.py` — все операции производства  
- `models/exceptions.py` — собственные исключения  
- `main.py` — CLI для взаимодействия с системой
